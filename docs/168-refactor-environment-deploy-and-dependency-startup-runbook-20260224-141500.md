# Refactor Environment Deploy and Dependency Startup Runbook (Externalized Config First)

## Scope
- Deploy and reboot the **refactor** container group only.
- Keep legacy stack in `/Users/alexb/Documents/Dev` unchanged.
- Supports the staged Cassandra setup:
  - `c1` (legacy-compatible rehearsal DB)
  - `c1-refactor` (upgrade target DB, e.g. Cassandra 4.1)

## Rule: Configuration Must Be Externalized
Do not hardcode hostnames/ports in service code or compose commands.

Primary config source:
- `/Users/alexb/Documents/Dev/Dev_new/gs-server/deploy/config/cluster-hosts.properties`

Generated/synced outputs (do not edit directly unless debugging):
- `/Users/alexb/Documents/Dev/Dev_new/gs-server/deploy/docker/refactor/.env`
- `/Users/alexb/Documents/Dev/Dev_new/gs-server/deploy/docker/refactor/.cluster-hosts.env`
- `/Users/alexb/Documents/Dev/Dev_new/gs-server/game-server/web-gs/src/main/resources/cluster-hosts.properties`
- `/Users/alexb/Documents/Dev/Dev_new/gs-server/deploy/docker/configs/.env`

Sync command after any host/port change:
```bash
cd /Users/alexb/Documents/Dev/Dev_new
/Users/alexb/Documents/Dev/Dev_new/gs-server/deploy/scripts/sync-cluster-hosts.sh
```

## Dependency Order (Refactor Stack)
Startup order for reliable boot:
1. `c1-refactor` (or `c1`, depending on active `CASSANDRA_HOST`)
2. `zookeeper`
3. `redis`
4. `kafka`
5. app services (`config-service`, `session-service`, `gameplay-orchestrator`, `wallet-adapter`, `bonus-frb-service`, `history-service`, `multiplayer-service`, `protocol-adapter`)
6. `mp`
7. `gs`
8. `static`

## Initial Deploy (Refactor Group)
```bash
cd /Users/alexb/Documents/Dev/Dev_new/gs-server/deploy/docker/refactor

# Build images that carry startup scripts/config glue
docker compose -p refactor --env-file .env build gs mp static

# Base infra
docker compose -p refactor --env-file .env up -d c1-refactor zookeeper redis

# Kafka (may need a short retry window after zookeeper restart)
docker compose -p refactor --env-file .env up -d kafka

# App services
docker compose -p refactor --env-file .env up -d \
  config-service session-service gameplay-orchestrator wallet-adapter bonus-frb-service \
  history-service multiplayer-service protocol-adapter mp gs static
```

## Reboot Test Requirement (Old `c1` Off, `c1-refactor` On)
Use this when validating failover/switch:

```bash
cd /Users/alexb/Documents/Dev/Dev_new/gs-server/deploy/docker/refactor

# Ensure old refactor c1 is OFF
docker compose -p refactor --env-file .env stop c1 || true

# Recreate all services except c1 (active Cassandra host comes from .env generated by cluster-hosts sync)
docker compose -p refactor --env-file .env up -d --build --force-recreate \
  redis bonus-frb-service gameplay-orchestrator history-service zookeeper kafka c1-refactor \
  mp session-service gs static wallet-adapter config-service multiplayer-service protocol-adapter
```

If Kafka exits with transient ZooKeeper broker-id `NodeExists` after restart, wait ~20-30s and rerun only Kafka:
```bash
docker compose -p refactor --env-file .env up -d kafka
```

## Verify GS/MP Are Using the Externalized Cassandra Host
```bash
cd /Users/alexb/Documents/Dev/Dev_new/gs-server/deploy/docker/refactor

docker compose -p refactor --env-file .env exec -T gs sh -lc '
  echo GS_ENV:$CASSANDRA_HOST:$CASSANDRA_PORT;
  for f in /var/lib/jetty/webapps/ROOT/WEB-INF/classes/ClusterConfig.xml \
           /var/lib/jetty/webapps/ROOT/WEB-INF/classes/SCClusterConfig.xml \
           /var/lib/jetty/webapps/ROOT/WEB-INF/classes/BigStorageClusterConfig.xml; do
    echo ---$f; grep -n "<hosts>" "$f";
  done'

docker compose -p refactor --env-file .env exec -T mp sh -lc '
  echo MP_ENV:$CASSANDRA_HOST:$CASSANDRA_PORT;
  grep -n "<hosts>" /tmp/web-mp-casino/WEB-INF/classes/mp-keyspace-config.xml'
```

Expected after switch:
- `<hosts>c1-refactor:9042</hosts>` in GS cluster XMLs
- `<hosts>c1-refactor:9042</hosts>` in MP keyspace config
