Guided Workflows

1) Add New Subcasino + New Bank (with Cassandra persistence)

This is the canonical flow to onboard a new bank under a subcasino.

  • Open /support/cache/bank/common/subcasinoSelect.jsp.
  • Create or prepare IDs via /support/newBankNSubCasino.do and /support/BankNSubCasinoControl.do.
  • Create the subcasino/bank via forms backed by /support/newSubCasino.do, /support/addbank.do, and optional /support/createFreeBank.do.

What happens in backend: bank creation action persists bank config through Cassandra bank persister and pushes cache refresh notifications to all GS nodes.

2) Add New Game to Existing Bank (writes game configuration)

  • Open bank details: /support/bankInfo.do?bankId=<BANK_ID>.
  • Launch add-game flow through /support/awayFromBankInfo.do and /support/cache/bank/common/addGame.jsp.
  • Submit create action: /support/createNewGame.do.

What happens in backend: game is created using base-game helper, persisted through remote-call persistence path, and propagated as refresh/invalidation events across GS servers.

3) Visualize Bank/Subcasino Information

  • Subcasino bank matrix: /tools/subCasinoInfo.jsp?subCasinoId=<ID>.
  • Full bank property dump: /tools/bankProperties.jsp?bankId=<ID>.
  • Specific property across banks: /support/showBanksDetailsProp.jsp?bankId=271|6274&prop=COMMON_WALLET_WAGER_URL.

These pages are the easiest visual tools for protocol type, API endpoints, currencies, master-bank relationship, and property-level comparisons.

Why Many Tools Look Broken

Audit Snapshot

Safe probe set (excluding shutdown trigger endpoint): 194 endpoints.

  • 200 OK: 115 endpoints
  • 500: 74 endpoints
  • 404: 2 endpoints
  • Timeout/Long run: 3 endpoints

Main Failure Causes

  • Direct open of action endpoints without required params or form context (example: /support/bankSelectAction.do needs bankId).
  • Internal JSPs intended as sub-views within a flow, not standalone pages.
  • Legacy/stale links (example: /support/templateManager/index.jsp returns 404).
  • Operational scripts that are slow or side-effect heavy, not normal UI pages.

Critical Safety Note

Do not open /support/prepareToShutdown.jsp during discovery scans.

This endpoint marks ShutdownFilter down and forces global 503 Service Unavailable until GS restart.

Notes

  • On local stack, links are rendered with http://localhost:81; internal container-level URLs may use port 8080.
  • Some JSP tools are low-level maintenance scripts and can alter live cache/config immediately; use in controlled environments.
  • For bank property editing, call /support/bankSupport.do first, then /support/bankSelectAction.do?bankId=<id> with a valid bank id.
  • For discovery scans, exclude /support/prepareToShutdown.jsp; it intentionally switches GS into shutdown mode (global 503 until restart).
  • Non-200 does not always mean broken business logic; many endpoints are action handlers that expect POST payloads, query parameters, or prior form/session context.
