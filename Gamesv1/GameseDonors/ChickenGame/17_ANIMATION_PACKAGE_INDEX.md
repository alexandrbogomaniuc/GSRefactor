## Engineering implications

- The donor presentation is event-driven. It is not just static atlas swaps.
- Reel stops, topper escalation, fire/lightning, coin transfer, and win overlays each have distinct packages and should be separate controllers in our original implementation.
- Reusing the donor state names directly in code is not required. Reusing the state shape is the useful part.
