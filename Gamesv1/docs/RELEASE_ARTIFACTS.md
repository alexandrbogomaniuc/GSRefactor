# RELEASE_ARTIFACTS

Deterministic release artifact set generated by:

```bash
corepack pnpm run release:pack -- --game <gameId>
```

Canonical contract source:
- `docs/gs/release-registration-contract.md`
- `docs/gs/fixtures/release-registration.sample.json`
- `docs/gs/schemas/release-registration.schema.json`

## Output location

- `games/<gameId>/release-packs/<releaseVersion>+<gitSha8>/`

Generated output is gitignored.

## Required files

- `artifacts/release-registration.json`
- `artifacts/client-artifact-manifest.json`
- `artifacts/asset-manifest.json`
- `artifacts/localization-manifest.json`
- `artifacts/math-package-manifest.reference.json`
- `artifacts/capability-profile.reference.json`
- `artifacts/compatibility-metadata.json`
- `artifacts/package-versions.json`
- `artifacts/release-metadata.json`
- `artifacts/integrity-metadata.json`
- `checksums.sha256.json`
- `GS_REGISTRATION_PACK.md`
- `ROLLBACK_PACK.md`
- `CANARY_CHECKLIST.md`
- `SMOKE_TEST_CHECKLIST.md`
- `RELEASE_PACK_SUMMARY.json`

## Registration schema expectations

`artifacts/release-registration.json` must use schema `slot-release-registration-v1` and include:

1. Entry URLs and manifest references.
2. `mathPackageManifestReference` (reference to immutable `math-package-manifest.json`).
3. Capability profile hash/reference.
4. Compatibility metadata reference.
5. Integrity fields (`hashAlgorithm`, checksums reference/hash, signature status/reference).
6. Canary + rollback metadata fields.
7. No non-schema fields; artifact shape is validated against `docs/gs/schemas/release-registration.schema.json`.

## Rules

1. No launch-time math injection.
2. Static assets and math artifacts are separate concerns.
3. Same commit + version produces deterministic output.
4. No secrets in generated files.
5. Do not treat `gs/template-params.json` as the final math artifact model for release packs.
