# RELEASE_ARTIFACTS

This document defines the deterministic release-pack artifact set generated by:

```bash
npm run release:pack -- --game <gameId>
```

## Goals

1. Immutable, versioned releases.
2. No launch-time math injection.
3. GS registers known package versions.
4. Browser assets served from CDN/static origin.
5. Deterministic/reproducible artifacts.

## Output Location

For each run:

- `games/<gameId>/release-packs/<releaseVersion>+<gitSha8>/`

Includes:

- `artifacts/client-bundle.manifest.json`
- `artifacts/asset.manifest.json`
- `artifacts/localization.manifest.json`
- `artifacts/math-package.manifest-reference.json`
- `artifacts/package-versions.json`
- `artifacts/release-metadata.json`
- `artifacts/gs-compatibility.json`
- `checksums.sha256.json`
- `GS_REGISTRATION_PACK.md`
- `ROLLBACK_PACK.md`
- `CANARY_CHECKLIST.md`
- `SMOKE_TEST_CHECKLIST.md`
- `RELEASE_PACK_SUMMARY.json`

## Required Artifact Semantics

### 1) Client Bundle Manifest

- Lists built client files from `dist/`
- Includes bytes + SHA-256 per file
- Includes entrypoint reference

### 2) Asset Manifest

- References source asset manifest (`src/manifest.json`)
- Lists runtime static asset files with checksums
- Separates asset concerns from math concerns

### 3) Localization Manifest

- Lists language folders and locale files
- Includes checksums per localization file

### 4) Math Package Manifest Reference

- References static math package artifact location (`math/math-pack.json` if present)
- Fallback reference may point to `gs/template-params.json`
- Explicitly records `launchTimeInjection=false`

### 5) Package Versions

- Records root/game/workspace package versions used in release

### 6) Release Metadata

- Records release id, game id, version, git SHA, deterministic timestamp source

### 7) GS Compatibility Metadata

- Runtime target metadata (GS HTTP runtime)
- Compatibility capabilities required by GS
- CDN/static URL pointers

### 8) Checksums

- SHA-256 list for all generated JSON artifacts

## Determinism Rules

1. Artifact files are written with stable key ordering.
2. File listings are lexicographically sorted.
3. Timestamp is deterministic:
- uses `SOURCE_DATE_EPOCH` when provided
- otherwise uses git commit date
4. Same input commit + version => same artifact content.

## No-Secrets Rule

Generated artifacts must not include:
- credentials
- tokens
- secrets
- private keys

## Command Examples

```bash
# Standard run (includes build + vite build)
npm run release:pack -- --game premium-slot --static-origin https://cdn.example.com/games

# Explicit release version
npm run release:pack -- --game premium-slot --version 1.0.0 --static-origin https://cdn.example.com/games

# Use existing build output
npm run release:pack -- --game premium-slot --skip-build --static-origin https://cdn.example.com/games
```
