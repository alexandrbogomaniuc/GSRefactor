# RELEASE_ARTIFACTS

Deterministic release artifact set generated by:

```bash
corepack pnpm run release:pack -- --game <gameId>
```

Canonical contract source:
- `docs/gs/release-registration-contract.md`

## Output location

- `games/<gameId>/release-packs/<releaseVersion>+<gitSha8>/`

Generated output is gitignored.

## Required files

- `artifacts/client-bundle.manifest.json`
- `artifacts/asset.manifest.json`
- `artifacts/localization.manifest.json`
- `artifacts/math-package.manifest-reference.json`
- `artifacts/package-versions.json`
- `artifacts/release-metadata.json`
- `artifacts/gs-compatibility.json`
- `artifacts/registration-artifact.json`
- `checksums.sha256.json`
- `GS_REGISTRATION_PACK.md`
- `ROLLBACK_PACK.md`
- `CANARY_CHECKLIST.md`
- `SMOKE_TEST_CHECKLIST.md`
- `RELEASE_PACK_SUMMARY.json`

## Rules

1. No launch-time math injection.
2. Static assets and math artifacts are separate concerns.
3. Same commit + version produces deterministic output.
4. No secrets in generated files.
5. Do not treat `gs/template-params.json` as the final math artifact model for release packs.
